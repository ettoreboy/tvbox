# Contributing

Contributions are welcome via **issues** and **pull requests**.

## How to contribute

1. **Open an issue** for bugs, ideas, or questions so we can discuss before you code.
2. **Fork the repo**, create a branch from `main`, and make your changes.
3. **Open a pull request** against `main`. Describe what you changed and link any related issue.
4. Keep changes focused; one PR per logical change is easier to review.

## Pull requests

- Branch off `main` and target `main`.
- Scripts are Bash; run them in a safe environment if you change behaviour.
- Docs: Markdown, same tone as the rest of the repo (clear, minimal).

Maintainers will review and may ask for tweaks. Once approved, your PR will be merged.

## Issues

Use **Issues** for:

- Bug reports (script fails, wrong behaviour)
- Feature requests or improvements
- Questions about usage or setup

Check existing issues first to avoid duplicates.

## Code of conduct

Be respectful and constructive. This is a small utility project; keep discussions on-topic and helpful.
